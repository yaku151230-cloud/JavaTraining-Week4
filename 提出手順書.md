# 内定者研修 課題提出手順書（PR 提出）

## はじめに
この文書は、研修課題を GitHub を利用して提出するための手順をまとめたものです。  
以下の手順に従って、**ブランチを切って実装 → PR（Pull Request）として提出**してください。

※ main ブランチへ直接実装・提出は行いません。

---

# 1. 作業用ブランチを作成する（必須）

作業は **必ず自分の main ブランチから新しいブランチを作成して行ってください。**

    git checkout main
    git pull origin main
    git checkout -b feature/JavaTraining-Week5

---

# 2. 指定されたファイルに実装を行う

課題内容および記述場所は、課題リポジトリの README に書かれています。

例：

- /src/main.py
- /src/index.js
- /src/Main.java

など、言語ごとに指定されたファイルへ実装してください。

※ 前回課題のコードを **コピーして作り直す必要はありません**。  
ブランチを切った時点で、前回までのコードはすでに含まれています。

---

# 3. 完成したら commit & push を行う

作業ブランチ（feature/weekX）で commit & push を行います。

    git add .
    git commit -m "WeekX 課題提出"
    git push origin feature/JavaTraining-Week5

---

# 4. PR（Pull Request）を作成する

1. GitHub 上の **あなたの Fork リポジトリ**へアクセス  
2. 画面上部に表示される **「Compare & Pull Request」** をクリック  
3. 以下のように設定します：

- Base（提出先）：研修担当者の課題リポジトリ main
- Compare（あなた側）：あなたのリポジトリの feature/JavaTraining-Week5

4. PR タイトルは以下の形式で記載してください：

    WeekX-課題提出(氏名)

例：

    JavaTraining-Week2-課題提出(提出太郎)

5. 本文には、以下を簡単に記載してください：
   - 今回対応した内容
   - 工夫した点
   - 不安な点（あれば）

6. **「Create Pull Request」** を押して提出完了です。

---

# 5. レビューへの対応

提出後、PR に対して研修担当者からレビューコメントが届きます。

修正が必要な場合は、**同じ作業ブランチ（feature/JavaTraining-WeekX）で修正**し、再度 commit & push してください。

    git add .
    git commit -m "レビュー指摘対応"
    git push origin feature/JavaTraining-WeekX

※ 新しい PR を作り直す必要はありません。

---

# 6. 提出完了後の流れ

- 全ての指摘が解消されたら、研修担当者が PR を **Close** します  
- PR は **マージしません**

その後、次の週の課題に進む前に、  
**レビュー済みの最終状態を自分の main ブランチへ反映してください。**

    git checkout main
    git reset --hard feature/JavaTraining-WeekX

---

# 7. 次の週の課題について

次の週の課題は、**更新済みの main ブランチから新しいブランチを切って開始**してください。

    git checkout -b feature/JavaTraining-Week(X+1)

---

## 注意事項
- main ブランチで直接作業しないでください
- コードのコピー＆ペーストによる再作成は不要です
- PR は「レビューの場」です。差分を意識してください
